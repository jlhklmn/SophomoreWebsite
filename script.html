Variables & Paths

Variables

To define a variable, you can use the word local keyword.

Creating variables allows you to organize your code better, as well as shorten code that may take up a lot of lines, such as functions.

local variableName1 = "This is a string value."


local variableName2 = function()

print(Functions can be stored in variables too!)

end

Variables does have rules of it's own. It cannot have a name that starts with a number, and it cannot be a reserved word. Variables are cammel cased, so Example and EXAMple are different variables.

Paths

Paths are used define objects under different parents, which can be found in the Explorer tab. Here is a quick example of how you would write the path to the Texture under Folder1.

local texture = game.Workspace.ExampleFolder.Folder1.Texture

-- Note that the folder 'ExampleFolder' is a folder object inside Workspace

We can see how we use the fullstop character to get children objects within another object. Here we can see that we started with the word 'game'. This keyword represents the whole game itself, and it is not visible in the explorer tab because everything inside the explorer is what's inside the keyword game. We can then see that we are navigating into 'Workspace', which you can find as the first object inside explorer. You can see how we used a dot (the fullstop) to connect these two keywords, which is basically grabbing the 'Workspace' object from game.

Using this logic, we can find other objects inside a previously defined object within one another, and we are able to navigate into the folder and find the texture object.
